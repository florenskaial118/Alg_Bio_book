7.2 Теоретические основы

7.2.1 Центральная догма молекулярной биологии

Краткое напоминание: ДНК \(\rightarrow\) РНК \(\rightarrow\) Белок.

7.2.2 Оценка сложности алгоритмов

Ключевой параметр при разработке и выборе инструментов. Алгоритмы выравнивания обычно имеют сложность O(n*m).

7.3 Попарное выравнивание последовательностей

7.3.1 Постановка задачи

Определить, насколько две последовательности похожи друг на друга.

  • Гэпы (пробелы): Вставляются для улучшения выравнивания, но за их появление вводятся штрафы, так как они нежелательны (эволюционно менее вероятны).

7.3.2 Глобальное выравнивание (Needleman-Wunsch)

Используется для сравнения полных последовательностей.

  • Принцип работы:
    1. Строится матрица (размером m x n).
    2. Каждая ячейка содержит оптимальный счет выравнивания подпоследовательностей.
    3. Счет вычисляется на основе трех соседних ячеек (диагональ, вверх, влево), что соответствует вариантам: совпадение/замена, открытие гэпа в первой последовательности, открытие гэпа во второй.
    4. Инициализация: Первый ряд и столбец заполняются штрафами за гэпы (сравнение нуклеотида с “пустотой”) - бывают линейными и аффиными.
  • Матрица замен:
    • Мы можем выбирать веса произвольно (например, +2 за совпадение, -2 за несовпадение).
    • Существуют стандартные матрицы, например, для нуклеотидов: +5 за совпадение, -4 за несовпадение.
    • Для аминокислот используются более сложные матрицы замен (PAM, BLOSUM).
  • Результат:
    • Счет выравнивания: Значение в правом нижнем углу. Это относительная величина, позволяющая сравнивать разные варианты выравнивания одной последовательности. Теоретический максимум: длина * макс. вес за совпадение.
    • Путь выравнивания: Восстанавливается обратным ходом от правого нижнего угла к левому верхнему.

7.3.3 Аффинный штраф за гэп

Линейный штраф (одинаковый за каждый гэп) биологически неточен. Эволюционно открыть гэп (инделу) сложно, но если он уже открыт, его расширение более вероятно.

  • Механизм: Используется аффинный штраф: Gap Penalty = G + L * E
    • G — штраф за открытие гэпа (большой, например -10).
    • E — штраф за продление гэпа (маленький, например -0.5).
  • Реализация: Требует двух дополнительных матриц для отслеживания состояния (открыт ли гэп в одной из последовательностей).

7.3.4 Локальное выравнивание (Smith-Waterman)

Используется для поиска похожих участков (доменов, фрагментов генов) внутри более длинных последовательностей (например, поиск гена в геноме).

  • Отличие от глобального: В матрицу вводится ноль. Если все варианты (диагональ, вверх, влево) дают отрицательный счет, в ячейку записывается 0.
  • Результат: Появляется возможность найти несколько небольших выравниваний с высоким счетом (локальных максимумов).
  • Обратный ход: Начинается от ячейки с максимальным счетом в матрице и продолжается, пока счет не упадет до 0.

7.4 Алгоритмы поиска в базах данных (BLAST)

Прямой перебор (попарное выравнивание запроса с каждой последовательностью в базе) слишком медленный.

7.4.1 Алгоритм FASTA

  1. Хеширование k-мер: Запрос и все геномы в базе разбиваются на короткие слова (например, тримеры для белков).
  2. Поиск совпадений: Строится хеш-таблица: ключ (k-мер) : значение (список позиций в геномах). Поиск по ключу происходит за O(1).
  3. Выявление диагоналей: Если в геномах и запросе есть одинаковые k-меры, вычисляется разница позиций w = i - j. Одинаковая разница w означает, что k-меры лежат на одной диагонали (без инделов).
  4. Затравка: Если на диагонали найдено больше трех совпадений подряд, она считается перспективной (затравкой).
  5. Расширение: Вдоль этой диагонали строится выравнивание (в узкой полосе, что снижает сложность до O(m*L)).

7.4.2 BLAST (Basic Local Alignment Search Tool)

Работает по схожему принципу, но использует более сложные стратегии поиска начальных совпадений (сидов) и их расширения.

  • Статистическая значимость (E-value):
    • Определение: E-value — это ожидаемое число случайных выравниваний с таким же (или лучшим) счетом, которое можно найти в базе данного размера.
    • Формула (концептуально): E-value = k * m * N * e^(-λ * Score)
      • N — размер базы данных.
      • m — длина запроса. Длинный запрос дает больше стартовых позиций для поиска совпадения.
    • Интерпретация: Чем меньше E-value, тем результат статистически значимее. Значение > 10^-6 обычно считается плохим (незначимым).
    • Важно: E-value зависит от размера базы данных N. Так как базы данных постоянно растут, E-value для одной и той же пары последовательностей может со временем уменьшаться.

7.5 Анализ прочтений (FASTQ)

  • Формат FASTQ: Стандартный формат для хранения прочтений (“ридов”) с секвенатора, включает качество каждого нуклеотида.
  • Препроцессинг: Удаление адаптеров и низкокачественных оснований.
  • Инструмент: trim-galore (обертка над cutadapt и FastQC).

7.6 Сборка генома de novo

7.6.1 Особенности

  • Бактерии: Собрать геном de novo относительно легко (множество инструментов).
  • Млекопитающие: Сборка сложна из-за размера и обилия повторов. Требуются длинные прочтения (long-reads).

7.6.2 Графы де Брёйна (de Bruijn graphs) — основа сборки

  1. Фрагментация: Все прочтения разбиваются на короткие k-меры (слова длиной k).
  2. Построение графа:
    • Вершины: (k-1)-меры (префиксы и суффиксы k-меров).
    • Ребра: k-меры.
  3. Поиск пути: Задача сборки сводится к поиску Эйлерова пути в ориентированном графе (пути, проходящего по каждому ребру ровно один раз).
  4. Проблема: Наличие повторов приводит к ветвлениям в графе, из-за чего Эйлеров путь может быть не единственным (или его сложно найти). Это приводит к фрагментации сборки на контиги.

7.6.3 Инструменты и практика

  • Популярный инструмент: megahit (хорошо подходит для сборки бактерий).
  • Команда:
megahit -r sample.fastq -t 8 -o output_directory
# -r : путь к fastq файлу (или пара файлов для парно-концевых чтений)
# -t : количество потоков CPU
# -o : директория для результатов
  • Результат: Файл final_contigs.fa, содержащий множество контигов (непрерывных участков). Из них для дальнейшего анализа обычно выбирают наиболее длинные и сопоставимые с целевой длинной сборки.

8 Лекция 2. Выравнивание на референсный геном

Задача: имея текст (геном) с ограниченным алфавитом (A, T, G, C), в котором могут быть повторы, нужно быстро найти в нём короткую последовательность (рид). Для этого текст индексируют.

Существует два основных подхода к индексации: на основе суффиксных массивов и на основе преобразования Барроуза — Уилера (BWT).

8.0.0.1 Алгоритм суффиксных массивов (на примере слова “banana$”)

Символ $ — терминатор, который лексикографически меньше любого другого символа (обозначает конец строки). Это нужно, чтобы корректно упорядочить суффиксы.

Шаги построения:

  1. Берём все суффиксы строки (окончания, начинающиеся с каждой позиции).
  2. Сортируем их лексикографически.
  3. Запоминаем начальные индексы отсортированных суффиксов.

Для “banana$” (индексация с 1)

  • Исходные позиции- 1-b, 2-a, 3-n, 4-a, 5-n, 6-a, 7-$
  • Суффиксы
    • 7 $
    • 6 a$
    • 4 ana$
    • 2 anana$
    • 1 banana$
    • 5 na$
    • 3 nana$

Суффиксный массив S_a = [7, 6, 4, 2, 1, 5, 3]

Как искать паттерн (например, “AN”): В отсортированном списке суффиксов все, начинающиеся с “AN”, будут стоять подряд. Находим левую и правую границу этого блока. Индексы суффиксов внутри блока (4 и 2) укажут на позиции в исходном тексте, где встречается паттерн.

Проблема: Алгоритм STAR использует суффиксные массивы, но не все вычислительные серверы имеют достаточно памяти для хранения полного суффиксного массива для больших геномов (например, человеческого). Поэтому был придуман более экономный подход — BWT.

8.0.1 Преобразование Барроуза — Уилера (BWT)

BWT — это перестановка символов исходной строки, которая группирует одинаковые символы вместе. Это позволяет выполнять поиск, используя меньше памяти.

Как получить BWT:

  1. Берём ту же таблицу циклических сдвигов (или суффиксов с $), что и для суффиксного массива.
  2. Сортируем строки лексикографически.
  3. Последний столбец полученной матрицы — это и есть BWT.

Для “banana$” отсортированные циклические сдвиги:

  1. $banana → последняя буква a
  2. a$banan → n
  3. ana$ban → n
  4. anana$b → b
  5. banana$ → $
  6. na$bana → a
  7. nana$ba → a

BWT("banana$") = "annb$aa"

8.0.1.1 Обратное преобразование BWT с помощью LF-отображения

LF-отображение (Last-First) показывает, какой символ из первого столбца соответствует данному символу из последнего столбца. Это позволяет восстановить исходную строку.

Массив LF = [2,6,7,5,1,3,4] означает, что символ из первой строки последнего столбца переходит в строку 2 первого столбца и т.д. Начиная со строки, где в последнем столбце стоит $, и идя по LF, можно восстановить исходную строку задом наперёд.

8.0.1.2 Два вспомогательных массива для быстрого поиска

Чтобы искать паттерн без полного восстановления текста, используют:

  1. Массив C (count): Для каждого символа алфавита (например, $, a, b, n) хранит количество символов в первом столбце BWT-матрицы, которые лексикографически меньше данного.
    • C($) = 0 (нет символов меньше $)
    • C(a) = 1 (только $ меньше a)
    • C(b) = 4 Это $,a,a,a
    • C(n) = 5 Это $,a,a,a,b
  2. Массив Occ (occurrence): Таблица, где для каждой строки (позиции в BWT) и каждого символа алфавита хранится, сколько раз этот символ встретился в BWT до текущей строки (или включая её). Это позволяет быстро считать, сколько раз символ встретился на префиксе BWT.

8.0.1.3 Поиск паттерна с помощью BWT (алгоритм backwards search)

Пусть мы ищем паттерн W = “ab”. Идём с конца паттерна к началу.

Обозначим low и high — границы диапазона строк в BWT-матрице, которые начинаются с текущего суффикса паттерна.

Формулы для шага, когда мы добавляем символ a слева к текущему паттерну W: - new_low(aW) = C(a) + Occ(a, low(W) - 1) + 1 - new_high(aW) = C(a) + Occ(a, high(W))

Где: - C(a) — сколько символов меньше a в первом столбце. - Occ(a, pos) — сколько раз символ a встретился в BWT на позициях от 1 до pos.

Изначально low = 1, high = N (длина текста) — пустой паттерн. Итерируемся по символам паттерна справа налево, обновляя границы. Если в какой-то момент low > high — паттерна нет.

Почему это хорошо: Не нужно хранить суффиксный массив целиком. Достаточно хранить BWT (строка длиной N) и таблицу Occ (можно сжимать). Это экономит память.

Ограничение: Классический BWT плохо работает при альтернативном сплайсинге, так как один рид может соответствовать разным комбинациям экзонов.

8.0.2 STAR — выравниватель для RNA-seq

STAR (Spliced Transcripts Alignment to a Reference) решает проблему сплайсинга.

Как работает: Имеется экзонно-интронная структура гена. Рид картируется на референс с помощью суффиксного массива. - Если рид целиком картируется на один экзон — хорошо. - Если часть рида не картируется (попадает на интрон), STAR пытается выровнять оставшуюся часть на другой экзон в пределах заданного окна.

STAR устойчив к адаптерам и мусорным последовательностям: если рид не картируется вообще, он его игнорирует.

Проблема нескольких изоформ: У одного гена может быть несколько вариантов сплайсинга (изоформ). Риды, картированные на общие экзоны, нельзя однозначно приписать одной изоформе.

8.0.2.1 Распределение ридов между изоформами (EM-алгоритм)

Пусть есть две изоформы с уникальными и общими экзонами. Простое отношение число ридов на уникальные экзоны(iso1) / число ридов на уникальные экзоны(iso2) не всегда корректно: у одной изоформы может быть просто больше экзонов.

Правильный подход — оценивать глубину покрытия (depth): количество ридов, приходящихся на единицу длины экзона. Но если уникальных экзонов нет, используют другой метод.

Аналогия с EM-алгоритмом (Expectation-Maximization):

Представьте, что у нас есть точки (риды) на прямой (геном), и мы знаем, что они порождены двумя распределениями (двумя изоформами). Мы не знаем параметры распределений и не знаем, какая точка от какого распределения.

Алгоритм:

  1. Инициализация: Задаём начальные параметры для двух распределений (например, два нормальных распределения A и B).
  2. E-шаг (Expectation): Для каждой точки xi вычисляем вероятность того, что она принадлежит распределению A или B, используя формулу Байеса:
    • P(A|xi) = P(xi|A) * P(A) / (P(xi|A)*P(A) + P(xi|B)*P(B))
    • P(B|xi) = 1 - P(A|xi) (Здесь P(A) и P(B) — априорные вероятности изоформ, часто равные).
  3. M-шаг (Maximization): Используя вычисленные вероятности как веса, пересчитываем параметры распределений (среднее, дисперсию) так, чтобы максимизировать правдоподобие данных.
  4. Повторяем шаги 2–3 до сходимости.

Этот принцип лежит в основе многих алгоритмов quantification (например, RSEM, Kallisto).

8.0.3 Алгоритм Kallisto (на основе графа де Брейна)

Kallisto работает без явного выравнивания (alignment-free).

  1. Построение графа де Брейна: Все возможные k-меры (например, тримеры) из референсного транскриптома становятся вершинами графа. Ребро — если два k-мера перекрываются на k-1 символ.
  2. Раскраска вершин: Каждая вершина (k-мер) красится в цвета соответствующих транскриптов, в которых он встречается.
  3. Квантификация: Рид разбивается на k-меры. Эти k-меры ищутся в графе. Набор цветов, в которые окрашены найденные k-меры, показывает, на какие транскрипты (или их комбинации) может выравниваться рид.
  4. Распределение: Используя EM-алгоритм, рид распределяется между транскриптами на основе цветов его k-меров.

Это очень быстро, потому что не нужно делать попарное выравнивание.

8.0.4 Контроль качества RNA-seq (RSeQC)

  • Clipping profile: Показывает процент ридов с обрезанными (clipped) участками разной длины. Много обрезанных участков может говорить о проблемах с качеством.
  • Gene Body Coverage: Равномерность покрытия генов ридами. Для свежезамороженных образцов покрытие равномерное, для парафиновых (FFPE) — распадается от 5’ к 3’ концу.
  • Распределение ридов: Соотношение ридов, упавших на экзоны, интроны и межгенные области. Высокий процент интронных ридов может указывать на загрязнение геномной ДНК или несплайсированный пре-мРНК.

8.0.5 Нормализация и дифференциальный анализ (DESeq2)

После получения матрицы “риды на ген” для разных образцов нужно решить проблему разной глубины секвенирования и разной длины генов.

8.0.5.1 Метрики нормализации

  • RPKM (Reads Per Kilobase per Million): (ri / li) / (сумма(rj) / 1e6) * 1e9. Нормализует на длину гена (li) и на глубину библиотеки. Но RPKM не подходит для сравнения между образцами, так как сумма RPKM в образце не константа.
  • TPM (Transcripts Per Million): (ri / li) / (сумма(rj / lj)) * 1e6. Нормирует так, что сумма TPM в образце всегда равна 1 миллиону. Лучше для сравнения внутри образца, но не между образцами с разным распределением экспрессии.
  • CPM (Counts Per Million): ri / сумма(rj) * 1e6. Нормализует только на глубину. Бесполезна для сравнения генов разной длины, но может использоваться для промежуточных расчётов.

Важное замечание про TPM: Если в контроле экспрессия генов A и B одинакова, а в эксперименте у A выросла, а у B не изменилась, то при TPM нормировке доля B в эксперименте может искусственно уменьшиться (так как “миллион” делится между всеми генами). Поэтому сравнивать TPM между разными образцами рискованно.

8.0.5.2 DESeq2

DESeq2 использует модель отрицательного биномиального распределения (подходит для счётных данных с большим количеством нулей).

Size factors: DESeq2 сначала вычисляет size factor для каждого образца — это медианное отношение каждого гена к среднему геометрическому по этому гену во всех образцах. На этот factor делят исходные counts. Это более робастная нормализация, чем просто на сумму.

Основные шаги в R:

  1. Создать объект DESeqDataSet (dds) из матрицы counts.
  2. Выполнить dds <- DESeq(dds). Эта команда делает три вещи:
    • Оценивает size factors.
    • Оценивает дисперсию для каждого гена.
    • Применяет shrinkage (сжатие) дисперсии: гены с низкой экспрессией имеют высокую дисперсию из-за шума; shrinkage “подтягивает” их дисперсию к среднему, что уменьшает ложные срабатывания.
  3. Важно: Использовать сырые counts на входе, а не нормализованные. Нормализация происходит внутри DESeq2.
  4. Для визуализации: vst(dds, blind=TRUE) (variance stabilizing transformation) — стабилизирует дисперсию, чтобы сделать данные пригодными для PCA и heatmaps.

Анализ:

  • PCA показывает, кластеризуются ли образцы по условиям (например, нормоксия vs гипоксия, разные клеточные линии).
  • Результаты: results(dds) — таблица с log2 fold change, p-value, padj (скорректированный на множественные сравнения, например, методом Бенджамини — Хохберга).
  • Enrichment analysis: Список дифференциально экспрессированных генов проверяется на принадлежность к определённым путям (KEGG, GO). Это помогает сделать биологический вывод (например, “гены из пути гликолиза обогащены в гипоксии”).

Примечание про аннотацию: Идентификаторы из Ensembl (ENSG00000…) неудобны для чтения. Их нужно заменять на символьные названия генов (HGNC) и типы генов (protein-coding, lncRNA) с помощью отдельной аннотации.

Cell Ranger — основной инструмент от 10x Genomics для первичной обработки single-cell данных.

Особенности:

  • Это коммерческий продукт с хорошей документацией и активной поддержкой
  • Рекомендуется к использованию, особенно для начинающих

9.2.1 Основные команды

  2. Запуск выравнивания

9.2.2 Выходные файлы

9.2.3 Проблема пустых капель и дуплетов

В идеале каждая капля содержит ровно одну клетку со специфической химией. Реальность:

  • Пустые капли (нет клетки) — частое явление. Cell Ranger автоматически их фильтрует и выдаёт два набора данных: сырой и отфильтрованный.
  • Дуплеты (2+ клетки в капле) — до 15% в некоторых экспериментах. Их нужно удалять отдельно.

9.2.4 Обнаружение дуплетов

Scrublet — популярная библиотека для обнаружения дуплетов:

  1. Симулирует дуплеты (объединяет случайные пары клеток)
  2. Вычисляет для каждой реальной клетки “doublet score” — насколько она похожа на симулированные дуплеты
  3. Клетки с высоким скором помечаются как дуплеты

Признак дуплета: нет уникальных высокоэкспрессируемых генов, вместо этого — умеренная экспрессия многих обычных генов.

Альтернативы: DoubletFinder, scDblFinder.

9.2.5 Альтернатива Cell Ranger: Kallisto bustools

  • Смотрит распределение ридов по изоформам
  • Работает быстрее, лучше находит паралоги
  • Не возвращает выравнивание
  • Требует ручной настройки координат баркодов

Важно: Cell Ranger не работает с химией других производителей (не 10x). Поэтому нужны альтернативы.

9.3 Контроль качества: что смотреть в web_summary.html

  • Количество клеток (можно задать вручную)
  • Медианное количество ридов на клетку — должно быть достаточным для полногеномного анализа
  • Общее количество ридов — 200-300 миллионов считается хорошим
  • Процент замапленных ридов на геном — минимум 40% должно попадать на экзоны (попадание на интроны допустимо)
  • Распределение UMI counts — должно иметь характерную форму “колена” (knee plot)

9.3.1 Пояснение терминов

  • Баркод (barcode) — уникальная последовательность, указывающая, из какой клетки пришла РНК
  • UMI (Unique Molecular Identifier) — уникальная метка для каждой исходной молекулы РНК, позволяет отличать реальные транскрипты от продуктов амплификации (PCR duplicates)

9.4.4 Нормализация в single-cell

Важное отличие от bulk: в single-cell нас интересует сравнение экспрессии между клетками, а не между генами внутри клетки. Поэтому TPM и другие метрики, нормализующие на длину гена, здесь не имеют смысла.

Стандартный подход (дельта-метод):

  1. Нормализуем каунты на коэффициент s — сумму всех каунтов в клетке, делённую на 1 млн
  2. Берём логарифм: expression = log(y/s + 1) 
# Нормализация на 10,000 каунтов на клетку и логарифмирование
sc.pp.normalize_total(adata, target_sum=1e4)
sc.pp.log1p(adata)

Если стандартный подход не работает — нужно обратиться к литературе и попробовать другие методы (например, SCTransform).

9.5 Сравнение клеточных типов между условиями

Если есть несколько временных точек или условий и нужно сравнить один клеточный тип, обычные DEG-алгоритмы могут быть не специфичны.

Решение: метод псевдо-булка (pseudo-bulk)

  1. Агрегируем все клетки одного типа в каждом образце в один “bulk” профиль
  2. Сравниваем эти псевдо-булки с помощью DESeq2 как обычные bulk RNA-seq образцы
  3. Это учитывает вариацию между образцами (репликами) и даёт более надёжные результаты

9.6 Batch-эффекты

При объединении данных из разных экспериментов (разные дни, разные протоколы) возникает batch-эффект — техническая вариация, маскирующая биологический сигнал.

Методы коррекции: - Harmony - scVI - BBKNN - Combat

Нельзя просто конкатенировать данные и нормализовать их вместе — batch-эффект нужно явно моделировать.

10 Семинар 4. Продвинутые методы анализа single-cell данных

10.1 Повторение: стандартный пайплайн анализа

Типовой порядок действий при анализе single-cell данных:

  1. QC (контроль качества)
  2. Doublet Detection (обнаружение дуплетов)
  3. Dispersion control / Feature Selection (отбор высоковариабельных генов)
  4. Scaling (масштабирование)
  5. PCA (главные компоненты)
  6. kNN (граф k ближайших соседей)
  7. UMAP / t-SNE (визуализация)

Важное замечание: Не все эти этапы одинаково критичны. Если в данных есть серьёзные проблемы, вы их увидите на визуализациях. Если данные хорошие — всё построится само собой. Но batch-эффекты могут всё испортить.

10.2 Проблема batch-эффектов

10.2.1 Почему это проблема?

Одна плашка (lane) секвенатора позволяет получить примерно 10 тысяч одиночных клеток. Современные когортные исследования могут включать до 6 миллионов клеток крови из агрегированных открытых данных.

При работе с множеством образцов возникает batch-эффект — техническая вариация, не связанная с биологией.

Источники batch-эффекта:

Источник Описание
Биологическая вариабельность Разные виды, разные доноры
Разная подготовка библиотек Нельзя просто конкатенировать данные
Разная химия 10x v2, 10x v3 (3-я версия лучше ловит низкоэкспрессированные гены)
Разные операторы, дни Человеческий фактор

10.2.2 Стратегии коррекции batch-эффектов

10.2.2.1 Вариант 1. Feature selection по отдельности (малоэффективный)

Для каждого образца отдельно: 1. Проводим стандартную обработку 2. Делаем feature selection (выбираем высоковариабельные гены) 3. Берём пересечение наборов генов из разных образцов

Проблема: теряется много информации, метод не решает проблему систематических сдвигов.

10.2.2.2 Вариант 2. BBKNN (Batch Balanced kNN)

Идея: строим граф k ближайших соседей, но поощряем рёбра не только между клетками одного batch, но и между разными batch. Это сближает клетки из разных партий, которые похожи по транскриптомному профилю.

Параметры:

  • Количество соседей (k) — задаётся
  • Разрешение (resolution) — влияет на количество кластеров

Реализация: sc.external.pp.bbknn в Scanpy.

10.2.2.3 Вариант 3. Harmony (рекомендуемый)

Алгоритм (упрощённо):

  1. Строим кластеры на основе центроидов клеток в PCA-пространстве
  2. Внутри каждого кластера определяем, из каких batch пришли клетки
  3. Итеративно “стягиваем” центроиды клеток из разных batch друг к другу
  4. Повторяем, пока изменения не станут незначительными

Реализации:

  • scanpy.external.pp.harmony_integrate (Python)
  • Seurat (R) — исторически сложилось, что Harmony появилась там первой

Важное замечание: Всегда остаётся вопрос: наблюдаемый batch-эффект — это систематическая ошибка или реальные экспериментальные данные? Иногда биологические различия (например, разные типы клеток) могут выглядеть как batch-эффект.

10.2.3 Детали реализации

UMAP — нелинейный метод снижения размерности. Важно: компоненты UMAP не имеют прямой биологической интерпретации. Не стоит делать биологические выводы из того, как именно расположились клетки на UMAP — этот метод нужен только для визуализации и формирования гипотез.

10.3 ScVI: автоэнкодеры для single-cell

10.3.1 Общая идея автоэнкодеров

Автоэнкодер — это нейросеть, которая учится сжимать данные в скрытое (латентное) пространство, а затем восстанавливать их обратно.

Структура:

  • Энкодер (encoder) — сжимает входной вектор в латентное представление
  • Декодер (decoder) — восстанавливает из латентного представления исходный вектор

Обучение: сравниваем исходный вектор с восстановленным, минимизируем ошибку восстановления.

Математика одного нейрона:

выход = a0 * w + b

где:

  • a0 — значение от предыдущего нейрона
  • w — вес ребра
  • b — bias (смещение)

10.3.2 ScVI для single-cell

Вход: вектор экспрессии генов в клетке (размерности n_genes)

Что делает ScVI:

  1. Энкодер превращает разреженный вектор экспрессии в плотное латентное пространство (аналог PCA, но нелинейный)
  2. Латентное пространство имеет вероятностную природу — каждый параметр — это распределение
  3. Декодер восстанавливает вектор экспрессии
  4. В процессе обучения модель учитывает batch-принадлежность клетки

Результат: скорректированные значения экспрессии (corrected expressions), очищенные от batch-эффекта.

Позиционирование: ScVI считается “overkill” для простых задач — мощный, но тяжёлый метод. Используется, когда другие методы не справляются.

10.4 Canonical Correlation Analysis (CCA) и Seurat

10.4.1 Математическая идея CCA

Пусть есть два набора данных:

  • X размера n1 × g (первые образцы, g генов)
  • Y размера n2 × g (вторые образцы, g генов)

CCA ищет такие векторы a и b, чтобы канонические переменные u = X^T a и v = Y^T b имели максимальную корреляцию:

\[\operatorname{corr}(u, v) = \frac{\operatorname{cov}(u, v)}{\sqrt{\operatorname{var}(u)} \cdot \sqrt{\operatorname{var}(v)}}\]

Матрица ковариации Σ_{xy} показывает взаимосвязи между генами в X и Y. Если Σ_{xy} > 0 — есть положительная связь.

Финальное уравнение:

\(\Sigma_{xx}^{-1} \Sigma_{xy} \Sigma_{yy}^{-1} \Sigma_{yx} a = \lambda a\)

где:

  • \(\Sigma_{xx}\) — ковариационная матрица внутри \(X\)
  • \(\Sigma_{yy}\) — ковариационная матрица внутри \(Y\)
  • \(\Sigma_{xy}\) — ковариационная матрица между \(X\) и \(Y\)
  • \(\Sigma_{yx} = \Sigma_{xy}^\top\) — транспонированная матрица
  • \(\lambda\) — собственное значение (максимизируемая корреляция)
  • \(a\) — искомый собственный вектор

10.4.2 Seurat CCA с якорями (anchors)

Seurat (R-пакет) использует CCA для поиска “якорей” — пар клеток из разных batch, которые являются взаимными ближайшими соседами (mutual nearest neighbors, MNN).

Оценка: работает чуть быстрее Harmony, но замешивает данные чуть хуже.

10.5 RNA velocity: траектории развития клеток

10.5.1 Основная концепция

На UMAP мы видим разные типы клеток. RNA velocity позволяет определить направление дифференцировки — как молодые клетки превращаются в зрелые.

Ключевая идея: в транскриптомных данных можно различить несплайсированный (незрелый) и сплайсированный (зрелый) транскрипты. Соотношение между ними говорит о динамике.

10.5.2 Модель кинетики транскрипции

Система дифференциальных уравнений:

\[ \frac{du}{dt} = \alpha(t) - \beta(t) \cdot u(t) \]

\[ \frac{ds}{dt} = \beta(t) \cdot u(t) - \gamma \cdot s(t) \]

Где: - \(u(t)\) — количество несплайсированного пре-мРНК - \(s(t)\) — количество сплайсированной мРНК - \(\alpha(t)\) — скорость транскрипции (образования РНК) - \(\beta(t)\) — скорость сплайсинга - \(\gamma\) — скорость деградации

10.5.3 Ландшафт дифференцировки

У млекопитающих: ландшафт развития имеет физическую интерпретацию — “гора”, на склоне которой есть ложбины (нейробласты, клетки глии). Под действием сигнальных путей клетка “скатывается” по горе и становится зрелой.

У беспозвоночных: “лестница Барроуза” (Barroux). Плюрипотентная клетка движется по кругу, пока не понадобится организму. В нужный момент она начинает дифференцироваться через “горку” в другой круг. Но кроме горки есть “лесенка”, по которой можно вернуться обратно.

Интересный факт: Бессмертные медузы (Turritopsis dohrnii) умеют менять направление своей дифференцировки — возвращаться из зрелой стадии в полип. В теории на методы траекторий возлагают большие надежды для изучения таких процессов.

10.5.4 Реальность

На практике RNA velocity работает плохо. Шум в данных, сложность оценки параметров и нестабильность алгоритмов делают результаты трудноинтерпретируемыми.

10.5.5 Формат данных

RNA velocity использует файлы формата .loom (HDF5-based), которые содержат: - Каунты сплайсированных транскриптов - Каунты несплайсированных транскриптов - Амбарные (ambiguous) транскрипты

11.1.1 Типы технологий (4 подхода):

Тип Принцип Проблема
1. Sequence-based На чипе (слайде) есть лунки с реактивами. На чип кладут срез ткани → химия подготавливает РНК → секвенирование. Низкое разрешение

  • Минимизируем cut — число разрезанных ребер.
  • Используем функцию N = cut(U,V)/vol(U) + cut(U,V)/vol(V)
    • vol(U) = сумма весов всех ребер внутри U (вес = длина или сходство).

11.3 3. Мультимодальные омики: CITE-seq (РНК + поверхностные белки)

Что хотим: Узнать, какие поверхностные белки есть на клетке и их концентрацию.

11.3.1 Как это работает:

  1. Создаются синтетические РНК-баркоды, привязанные к антителам.
  2. Антитела прикрепляются к белкам на поверхности клетки.
  3. Клетку расщепляют.
  4. Секвенируют одновременно РНК и баркоды антител (выравнивание — раздельно).

11.3.2 Анализ белковых данных (ADT — Antibody-Derived Tags):

Отличия от РНК-анализа:

  • Нет выделения высоковариабельных генов (HGV).
  • PCA используется только для снижения шума (не для структуры).
  • Построение графа k ближайших соседей → кластеризация.
  • Визуализация: tSNE, UMAP.

Ключевой шаг: нормализация CLR (Centered Log Ratio)

clr(x) = [ log(x1)/g(x), ..., log(xn)/g(x) ]
где g(x) — среднее геометрическое.
  • Почему работает для белков: концентрация белка не может быть 0 (в отличие от РНК, где 0 — это норма).
  • Для РНК CLR нельзя: если экспрессия = 0, формула ломается.

11.3.3 Пример кода (без batch-коррекции):

mdata['ADT'].X = mdata['ADT'].layers['counts'].copy()
mu.prot.pp.clr(mdata['ADT'])          # CLR-нормализация
mdata['ADT'].var['highly_variable'] = True

sc.pp.scale(mdata['ADT'], max_value=10)
sc.tl.pca(mdata['ADT'], n_comps=30)
sc.pp.neighbors(mdata['ADT'], n_neighbors=20)

sc.tl.umap(mdata['ADT'])
sc.pl.umap(mdata['ADT'], color='batch', title='Batches, ADT')

Важно: UMAP для белков и UMAP для РНК могут выглядеть по-разному — это нормально (UMAP — стохастический алгоритм).

11.4 4. Как объединить разные модальности (РНК + белки) в один анализ?

Проблема: базисы (пространства признаков) белков и РНК сильно отличаются. Перенос кластеров из одного базиса в другой работает плохо.

11.4.1 Три метода:

Метод Идея Когда хорошо
1. MOFA + Harmony MOFA (мультиомиксный факторный анализ) → общий латентный базис → коррекция базиса через Harmony Данные сильно разные
2. wNN (weighted Nearest Neighbors) Строим два графа соседей (РНК и белки). Для каждой точки восстанавливаем профиль по соседям из другой модальности → ищем веса, по которым хорошо предсказывается другая модальность Гетерогенные данные
3. PCA concatenation Делаем PCA отдельно для каждой модальности → объединяем результаты Оказывается, работает хорошо и просто

Вывод: Неожиданно, но простое склеивание PCA часто работает достаточно хорошо.

12 Семинар 6. ChIP-seq и ATAC-seq: анализ пиков (связывание белков и открытость хроматина)

12.1 Часть 1. ChIP-seq (поиск мест связывания белков с ДНК)

12.1.1 1.1 Принцип метода

Что ищем: участки ДНК, с которыми связывается интересующий нас белок (транскрипционный фактор, гистоновая метка и т.д.).

Этапы эксперимента:

  1. Сшивка белка с ДНК (cross-linking).
  2. Иммунопреципитация — антитело «вылавливает» белок вместе с ДНК.
  3. Фрагментация ДНК (рестриктазы или ультразвук).
  4. Смыв белка, остаются только участки ДНК, которые были с ним связаны.
  5. Добавление адаптеров → секвенирование.
  6. На выходе fastq (но это ДНК, а не РНК).

Почему проще, чем RNA-seq: нет изоформ, сплайсинговариантов, меньше транскрипционных шумов.

12.1.2 1.2 Стандартный pipeline

Шаг Инструменты Что делаем
1. Контроль качества FastQC, fastp Оценка качества чтений
2. Выравнивание (mapping) Bowtie2, BWA На референсный геном
3. Фильтрация BAM samtools Убираем плохие выравнивания
4. Удаление дупликатов Picard MarkDuplicates PCR-дупликаты искажают пики
5. Peak calling MACS2 / MACS3 Определяем значимые пики
6. Downstream анализ MEME, IGV, DiffBind Мотивы, GO, визуализация

12.1.3 1.3 Особенности данных ChIP-seq

  • Парно-концевые чтения (paired-end) — лучше для оценки фрагментации.
  • Риды распределяются на плюс- и минус-цепи относительно места связывания белка.
  • Ключевая идея: старт транскрипции находится справа или слева от белка → это создаёт характерное распределение ридов с ложбинкой (а не пиком) в месте связывания.

12.1.3.1 Кросс-корреляция между плюс- и минус-цепью:

  • Позволяет оценить качество эксперимента (хорошая выборка → чёткий сдвиг).
  • Используется для peak calling (MACS ищет этот сдвиг).
  • Артефактный пик (иногда выше настоящего) — игнорируем, сравнивая с контролем (input DNA).

12.1.4 1.4 Пример команд (обработка данных)

# 1. Скачиваем хромосому (для теста)
wget https://hgdownload.soe.ucsc.edu/goldenPath/hg38/chromosomes/chr9.fa.gz
gunzip chr9.fa.gz

# 2. Скачиваем SRA результат
prefetch SRR094236
fasterq-dump --split-files SRR094236/SRR094236.sra

# 3. FastP (FastQC + фильтрация, быстрее классического FastQC)
fastp -i SRR094236.fastq -o SRR094236.fastp.fastq

# 4. Индексируем хромосому BWA
bwa index hg38/chr9.fa
bwa mem -t 32 hg38/chr9.fa SRR094236.fastp.fastq > CTCF.chr9.sam

# 5. SAM → BAM
samtools view -@ 32 -b CTCF.chr9.sam > CTCF.chr9.bam

# 6. Сортировка и удаление дупликатов
samtools sort -@ 32 -o CTCF.chr9.sorted.bam CTCF.chr9.bam
picard MarkDuplicates -I CTCF.chr9.sorted.bam -O CTCF.chr9.rmdup.bam \
    -M dedup_metrics.txt -VALIDATION_STRINGENCY LENIENT \
    -REMOVE_DUPLICATES true -ASSUME_SORTED true

# 7. QC выравнивания
samtools flagstat CTCF.chr9.rmdup.bam

# 8. Оценка кросс-корреляции (phantompeakqualtools)
Rscript run_spp.R -c=CTCF.chr9.rmdup.bam -savp=spp_ctcf.pdf -out=spp_metrics_ctcf.txt

12.1.5 1.5 Peak calling с MACS3

Вход: BAM ChIP-seq + BAM контроль (input DNA)
Выход: BED-файл с координатами пиков

macs3 callpeak -t CTCF.chr9.rmdup.bam \
               -c input.chr9.rmdup.bam \
               -f BAM -g 1.4e8 -q 0.05 \
               -n CTCF_peaks_q005

Что делает MACS (важно понять):

  1. Учитывает разную глубину секвенирования между образцом и контролем — усредняет покрытие.
  2. Удаляет избыточность (redundancy).
  3. Смещает риды на d/2 (половина средней длины фрагмента), потому что настоящий пик находится не там, где риды, а между плюс- и минус-цепями (ложбинка).
  4. Моделирует распределение ридов по Пуассону → p-value, q-value (FDR).

Типы пиков:

  • narrow peaks — для транскрипционных факторов (точное место связывания).
  • broad peaks — для гистоновых меток (области).

12.2.5 2.5 Нормализация: TF-IDF (из NLP)

  • Проблема: матрица 0/1, но редкие пики важнее частых.
  • Решение:
    • TF (term frequency) = число клеток с пиком t / общее число клеток в документе d.
    • IDF (inverse document frequency) = log(общее число клеток D / число клеток с пиком t).
    • TF-IDF = TF × IDF — взвешенная встречаемость пика.

12.2.6 2.6 Разложение матрицы: SVD (Singular Value Decomposition)

Математика:

X = U × Σ × V^T
  • X — исходная матрица (клетки × пики).
  • U — веса клеток по компонентам.
  • Σ (диагональная) — важность каждой компоненты (сингулярные числа).
  • V — вклад пиков в компоненты.

Главные компоненты = U × Σ

Важный шаг: выкидываем первую компоненту, потому что она коррелирует с глубиной секвенирования каждой клетки (технический артефакт).

Затем строим UMAP по оставшимся компонентам.

12.2.7 2.7 Интеграция с RNA-seq (перенос лейблов)

  • Нельзя рисовать ATAC и RNA на одном UMAP (разные эксперименты, разные базисы).
  • Можно: перенести лейблы клеточных типов с RNA на ATAC (например, через Seurat или Liger).

  • Ищем, какие регуляторные элементы работают в каких клетках.
  • Вопрос: какой транскрипционный фактор отвечает за открытость этого участка?

13 Семинар 7. AlphaFold: нейросеть для предсказания структуры белка

13.1 1. Что делает AlphaFold?

AlphaFold — это нейросеть, которая на вход получает последовательность аминокислот (АК) белка, а на выходе выдает его трехмерную структуру (координаты атомов).

Для каждой аминокислоты сеть вычисляет уверенность в правильности предсказанных координат:

  • Синий — высокая уверенность (структура предсказана хорошо).
  • Красный — низкая уверенность.

Важно: Альфа-спирали сеть «любит» и почти всегда маркирует их как хорошо определенные, даже если в реальности они могут быть гибкими.

13.2 2. Общий порядок работы AlphaFold

  1. Вход: последовательность аминокислот.
  2. Поиск гомологов:
    • Обращение к базе последовательностей (например, UniRef) → построение множественного выравнивания (MSA).
    • Обращение к базе структур (например, PDB) → поиск известных структур для гомологов.
      • Если структуры найдены → решение ускоряется (используется как шаблон).
      • Если данных нет → начальная пространственная структура генерируется случайно.
  3. Формирование двух матриц:
    • Матрица MSA (множественное выравнивание) — размер (число последовательностей) × (длина белка).
    • Матрица парных расстояний — для каждой пары АК хранится информация о предполагаемом пространственном расстоянии.
  4. Итеративное уточнение через блоки эволюционных трансформеров (48 раундов).
  5. Выход: предсказанные 3D-координаты + карта уверенности.

13.3 3. Эмбеддинги: как компьютер понимает аминокислоты

Компьютеры работают с числами. Каждую аминокислоту (или слово в тексте) кодируют в вектор — это называется эмбеддинг.

13.3.1 One-hot encoding — простейший способ

  • Каждая АК → вектор из 0 и 1, где 1 только на одной позиции.
  • Недостаток: нет смысловых связей между АК.

13.3.2 Смысловые эмбеддинги (например, из AlphaFold или Word2Vec)

  • Вектор кодирует контекст: последовательность, эволюционное происхождение, вид белка.

  • Сложение векторов сохраняет смысл:

    \[ \text{Король} - \text{Мужчина} + \text{Женщина} \approx \text{Королева} \]

  • В нашем случае в матрице не просто клетки, а вектора-эмбеддинги. Они хранят:

    • информацию о самой АК,
    • о ее положении в последовательности,
    • о виде организма,
    • о гомологах из MSA.

13.4 4. Механизм внимания (Attention) — основа всего

Это алгоритм, лежащий в основе всех современных генеративных моделей (GPT, BERT, AlphaFold).

13.4.1 4.1. Идея внимания

Каждое слово (или АК) кодируется вектором. Модель добавляет к нему контекст от других слов. При обучении модель пытается по контексту предсказать следующее слово.

13.4.2 4.2. Как это работает на примере слов

Пусть у нас есть три слова с эмбеддингами \(E_1, E_2, E_3\).

13.4.2.1 Шаг 1. Создаем запросы (Query) и ключи (Key)

  • Каждый \(E_i\) умножается на матрицу весов \(W_q\) → получаем вектор запроса \(Q_i\).
  • Каждый \(E_i\) умножается на матрицу весов \(W_k\) → получаем вектор ключа \(K_i\).

\[ Q_i = E_i \cdot W_q, \quad K_i = E_i \cdot W_k \]

13.4.2.2 Шаг 2. Строим матрицу внимания (Score matrix)

Строки — ключи \(K\), столбцы — запросы \(Q\). Перемножаем векторы скалярно. Если значение большое — между словами есть сильная смысловая связь.

\[ S_{ij} = Q_i \cdot K_j \]

13.4.2.3 Шаг 3. Применяем softmax к каждому столбцу

Превращаем оценки в вероятности (как сильно слово \(j\) должно «внимать» слову \(i\)):

\[ \text{softmax}(S)_{ji} = \frac{e^{S_{ji}}}{\sum_k e^{S_{jk}}} \]

13.4.2.4 Шаг 4. Получаем значения (Value)

Каждый \(E_i\) умножается на матрицу \(W_v\) → получаем вектор значения \(V_i\):

\[ V_i = E_i \cdot W_v \]

13.4.2.5 Шаг 5. Выход внимания

Для каждого \(i\) выходной вектор — взвешенная сумма \(V\) всех слов, где веса — вероятности из шага 3:

\[ \text{Output}_i = \sum_j \text{softmax}(S)_{ji} \cdot V_j \]

13.4.3 4.3. Визуализация матриц

  • Первая матрица (внимание Q-K): размер (число слов) × (число слов). Показывает связи между словами.
  • Вторая матрица (внимание к значениям): строки — выходы, столбцы — значения \(V\).

13.5 5. Внимание в AlphaFold (специфика)

В AlphaFold механизм внимания работает на двух уровнях:

13.5.1 5.1. Внимание по строкам MSA

  • Смотрим, как одна аминокислота в данной последовательности влияет на другую аминокислоту в той же позиции в другой последовательности (гомологе).
  • Улавливает коэволюцию: если при мутации в одной позиции меняется другая — значит, они пространственно близки.

13.5.2 5.2. Внимание по столбцам MSA (или «внимание к себе»)

  • Срез для отдельной аминокислоты.
  • Смотрим, как эта АК ведет себя в разных видах (в разных последовательностях выравнивания).
  • Помогает понять, какие позиции консервативны (важны для структуры).

13.5.3 5.3. Смысловая эволюция выравниваний

В процессе обучения сеть учится различать:

  • Консервативные позиции (меняются редко) — обычно важны для структуры.
  • Ковариирующие пары (когда изменение в одной позиции компенсируется изменением в другой) — сигнал о пространственном контакте.
  • Шум — неэволюционные совпадения.

13.6 6. Правило треугольника (Triangle Rule)

Это важное ограничение, отражающее физический смысл.

Для любых трех аминокислот \(A, B, C\) расстояния между ними должны удовлетворять неравенству треугольника:

\[ d(A, C) \leq d(A, B) + d(B, C) \]

  • AlphaFold проверяет это правило для всех троек АК и корректирует предсказания.
  • Проблема: вычислительно очень сложно (\(O(N^3)\)), поэтому иногда блоки с этим правилом убирают для ускорения.

13.7 7. Итоговая архитектура AlphaFold

  1. Вход: MSA + случайные координаты.
  2. Эмбеддинги: перевод АК и пар АК в векторы.
  3. 48 блоков эволюционного трансформера. В каждом блоке:
    • Внимание по строкам MSA (эволюционный контекст).
    • Внимание по столбцам MSA (консервативность/коэволюция).
    • Правило треугольника (пространственные ограничения).
    • Обновление матриц MSA и парных расстояний.
  4. Выходная голова: предсказание 3D-координат и уверенности.

13.8 8. Дополнительные технические детали

13.8.1 REINFORCE-шаг

После предсказания структура оптимизируется градиентным спуском, используя физический потенциал (энергия Розетты).

13.8.2 Ансамбль

AlphaFold запускается несколько раз с разными случайными инициализациями, затем результаты усредняются.

13.8.3 Контрольная лента (recycling)

Выход сети снова подается на вход (обычно 3–4 цикла) для улучшения.
